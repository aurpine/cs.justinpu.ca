Problem #2: Genetic Probability

This is a combinatorics problem. The overall probability is the combined probability of each pairs of the genes. So therefore, you just simply multiply the probabilities of the individual genes to find the over probability.

Since there are only 4 possible combinations of gene matching, we can just hard code the four possibilities to check if the combination matches the target gene. Since it is a combination and not a permutation ("aA" is the same as "Aa"), we have to check the different orders.

Since these are strings and only capital and non-capital letters of the same letter are in one gene, we can just add up the char values and check if the sum is equal to that of the target.

We find that n out of the four possibilities yield the target and we update the overall chance by multiplying by n / 4. 

import java.util.*;

class P2 {
    public static void main(String[] args) {
        Scanner in = new Scanner(System.in);
        
        int T = in.nextInt();
        for(int t = 0; t < T; t++) {
            int N = in.nextInt();
            String a = in.next(), // Parent 1
                b = in.next(),  // Parent 2
                c = in.next(); // Target offspring
            double prob = 1; // Total probability of target
            
            for(int n = 0; n < N; n++) {
                int pos = 0; // Probability of target gene (out of 4)
                int sum = c.charAt(n * 2) + c.charAt(n * 2 + 1); // Sum of target gene
                // The 4 possible combinations of genes
                if(a.charAt(n * 2) + b.charAt(n * 2) == sum)
                    pos++;
                if(a.charAt(n * 2) + b.charAt(n * 2 + 1) == sum)
                    pos++;
                if(a.charAt(n * 2 + 1) + b.charAt(n * 2) == sum)
                    pos++;
                if(a.charAt(n * 2 + 1) + b.charAt(n * 2 + 1) == sum)
                    pos++;
                prob *= pos / 4.;
            }
            System.out.println(prob);
        }
    }
}
