Problem 1: Pass or Fail

This is an implementation problem. To avoid floating-point errors, use the following formula: \(W_T \times T_i + W_A \times A_i + W_P \times P_i + W_Q \times Q_i \ge 5000\).

Complexity: \(O(N)\)

Problem 2: Spindie

The main trick to this problem is noting that the numbers on the spinner only go up to 100. After dealing with duplicate numbers, just brute force the 3 spins to check if it is possible.

Complexity: \(O(100^3)\)

Problem 3: Railway Sort

This problem is a greedy/implementation problem. We note that from right to left, all the numbers that we can count down from ~N~ do not ever need to be moved.
