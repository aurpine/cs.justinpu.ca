C++ has some differences with data types. These different primitive data types will be covered in this section.

Long Long Int

This type seems very verbose. It can be shortened to long long. Similarly, short int can be shortened to short. Why long long int versus long? There's a data type called long int, which uses 32 or 64-bit (int is 32-bit), depending on the system, while long long int is always 64-bit. For this reason, one should never use long int.

Unsigned Integral Numbers

C++ supports the use of unsigned numbers. Normal shorts, ints, longs, etc. are signed, meaning that they can have negative numbers-at the expense of having one less binary digit. Unsigned numbers can only hold non-negative numbers but can store a number two times higher than their signed counterpart. To declare an unsigned number, simply add the keyword before the data type. There are however only five unsigned data types. 

unsigned char a;
unsigned short int b; // Or just short
unsigned int c;
unsigned long int d; // Or long
unsigned long long int e; // Or long long

Long Double

This is a data type that I haven't mentioned yet. It isn't often used, since, it may or may not differ depending on the compiler used. Additionally, floating point (decimal numbers) generally aren't used in programming due to their inaccuracy. The number 0.1, cannot be expressed in binary, (similar to how one third is 0.33333333...), thus, it can become an odd number after a few operations such as 0.99999999999786599040.

Literals

Literals also exist in Java. There are some different literals for different data types. The following code illustrates these differences. 

short a = 1;
int b = 2;
long long int c = 123456789000ll; // Note the ll at the end
char d = 'A';
float e = 1.0;
double f = 1.00001;
long double g = 1.00000000000000001l; // Note the l
unsigned long long int g = 123456789000ull; // Note the ull
unsigned int h = 1u; // Note the u
int i = 0xFF; // 0x denotes hexadecimal
long long int j = 0xFFFFFFll; // Hexadecimal long
unsigned long long int j = 0xFFFFFFull; // Unsigned hexadecimal long
int k = 0127; // Octal, actual value is 87

Notice the LL added to the end of a long literal to permit the unusually long number, that wouldn't be normally valid for a regular sized variable. A u is added at the end to indicate an unsigned number. Similarly, the prefix 0x is added to a number to denote that it is in hexadecimal(base 16). The prefix 0 is added to denote octal (base 8). Any of these modifiers can be added to a number appriopriately.

Booleans

In C, there is no boolean data type, there are only ints. In C++, there is a type, called bool. It is like a combination of C style booleans and Java style booleans. In C, boolean values were expressed as 0 for false and 1 for true. In Java as you know, false is false and true is true. In C++, the bool data type accepts both types of literals. You are able to use a bool as an integer. 

bool a = true;
int b = 2 + a; // b now has a value of 3

Strings

Strings in C++ are usually annoying, which is why many prefer Java for string manipulation. C-style strings, are char arrays. These arrays work just like other arrays in C++. Usually, strings are initialized as follows: 

char word[6] = "hello";

The length of the char array is 6, one more than the number of letters in the string, because C++ requires the null character ('\0') to determine the end of the array/string. The above code is equivalent to: 

char word[6] = {'h', 'e', 'l', 'l', 'o', '\0'};

The null character doesn't have to be the last character of a char array. 

char word[10] = {'h', 'e', 'l', 'l', 'o', '\0'};

A longer array length allows longer strings to be used. This ensures that the string wouldn't be truncated. Declare a string similarly to a 2D array, set a maximum size, either stated in the problem, or a big number like 255 or 100.

As you can see, c style strings in C++ are quite complicated. That's why I prefer to use the C++ string. It's a built-in class with many features that Java has. This will be discussed in more detail later on.

Arrays

Arrays in C++ have the same concept as in Java, but the syntax is slightly different.

Arrays in Java

int[] a;
int[] b = new int[10];
int[] c = new int[3] {1, 2, 3};

Now the same code in C++

int a[];
int b[10];
int c[3] {1, 2, 3};

Arrays will be discussed in further detail in the Arrays lesson.

Final Variables

The final keyword equivalent in C++ is const, standing for constant. It is used the same way as in Java.
