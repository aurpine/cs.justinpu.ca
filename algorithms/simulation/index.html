Simulation

Simulation problems are where a process or situation is described, and as the solution, you make a program to simulate it. These problems are easy-the only challenge is implementing the solution.

Example

The Collatz Conjecture says that if you take any number \(n\) and repeat the process: if \(n\) is odd, multiply by 3 and add 1, and if it's even, divide by 2, you will eventually reach 1. Given a number \(N\), find how many iterations of this process it takes to get to 1.

The fastest and simplest way is to create a program to simulate this process and keep count of the number of iterations. 

int c== 0; // counter
while(N != 1) {
    if(N % 2 == 0) // If N is even
        N /= 2;
    else
        N = N * 3 + 1;
    c++;
}

Conclusion

These problems require you to code a pattern/process/situation described. Generally, output when a given condition is met, i.e. number of iterations. Output could also be a value at a certain point in time. These solutions all require the use of a loop and variables to store counters or terms.

These simulation problems vary and there's no specific way to solve them. Problem solving skills are required. You can train by attempting the practice problems below.
